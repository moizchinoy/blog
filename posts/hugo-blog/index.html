Creating Hugo Blog

Create a repository on Github (e.g. daily)

# Create Nix Shell
nix-shell -p hugo git

# Create hugo site (site name is daily)
hugo new site daily

# cd to daily directory
cd daily

# git initialization
git init

# Rename git branch
git branch -M main

# Adding public which is a publish directory to .gitignore
echo "public/" >> .gitignore

# Add hugo theme
git submodule add https://github.com/nanxiaobei/hugo-paper themes/paper

# Configure Hugo to use our theme
echo "theme = 'paper'" >> hugo.toml

# Add email and name to git config
git config user.email 'your email'
git config user.name 'your name'

# commit the changes
git commit -m "Initial Commit"

# Add remote git repository, instead of usual 'origin', we are using 'github', actually it can be anything
git remote add github <your git repo url>

# Push the changes
git push -u github main

To run the hugo server locally, execute following command

hugo server

To include draft pages, execute

hugo server -D

Execute following command to publish the site

hugo

Hugo creates the entire static site in the public directory in the root of your project. This includes the HTML files, and assets such as images, CSS files, and JavaScript files. When you publish your site draft, future, or expired content are not included.

To include draft content while publishing, run

hugo -D

By default the content is published in public directory. Use --destination flag to override it.

To add new content, execute following

hugo new content content/posts/my-first-post.md

The new content is in draft mode and is not published by default.

For configuring deployment on Github Pages, refer this page Host on GitHub Pages
