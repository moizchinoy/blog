Hugo Website in Azure Storage Container

Apr 23, 2024

In this post we will deploy the hugo website from Github to Azure Storage container.

After creating a Hugo Blog, lets deploy the website in Azure Storage container.

nix-shell -p azure-cli

az login

# Create resource group
az group create --name hugo-resource-group --location eastus

# Create storage account - replace <account-name> with a unique name 
az storage account create --name <account-name> --resource-group hugo-resource-group --location eastus --sku Standard_LRS --kind StorageV2 --allow-blob-public-access false

# Enable static website hosting - this will automatically create a container called `$web` container in the storage account which will contain files for our website
az storage blob service-properties update --account-name <storage-account-name> --static-website --404-document 404.html --index-document index.html

# Generate a credentail for Github Action
az ad sp create-for-rbac --name "hugo-github-action-sp" --role contributor --scopes /subscriptions/<subscription-id>/resourceGroups/hugo-resource-group --json-auth

Store the JSON output from the last command, we will need it in the next step.

Next create a secret in git repo in Github. Key should be AZURE_CREDENTIALS (name can be anything) and paster the JSON from the last step as value.

Create a Github Action and add following YAML:

name: Deploy Hugo site to Azure Blob Container

on:
  # Runs on pushes targeting the default branch
  push:
    branches: ["main"]

  # Allows you to run this workflow manually from the Actions tab
  workflow_dispatch:

# Default to bash
defaults:
  run:
    shell: bash

jobs:
  # Build job
  build:
    runs-on: ubuntu-latest
    env:
      HUGO_VERSION: 0.124.1
    steps:
      - name: Install Hugo CLI
        run: |
          wget -O ${{ runner.temp }}/hugo.deb https://github.com/gohugoio/hugo/releases/download/v${HUGO_VERSION}/hugo_extended_${HUGO_VERSION}_linux-amd64.deb \
          && sudo dpkg -i ${{ runner.temp }}/hugo.deb          
      - name: Install Dart Sass
        run: sudo snap install dart-sass
      - name: Checkout
        uses: actions/checkout@v4
        with:
          submodules: recursive
      - name: Build with Hugo
        env:
          # For maximum backward compatibility with Hugo modules
          HUGO_ENVIRONMENT: production
          HUGO_ENV: production
        run: |
          hugo \
            --minify          
      - name: Login to Azure
        uses: azure/login@v1
        with:
          creds: ${{ secrets.AZURE_CREDENTIALS }}
      - name: Upload to blob storage
        uses: azure/CLI@v1
        with:
          inlineScript: |
            az storage blob upload-batch --account-name <account-name> --auth-mode key -d '$web' -s ./public            
    
      # Azure logout
      - name: logout
        run: |
            az logout            
        if: always()
